### Componentes de un `StatefulSet`

```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx
  labels:
    app: nginx
spec:
  ports:
  - port: 80
    name: web
  clusterIP: None
  selector:
    app: nginx
---
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: web
spec:
  selector:
    matchLabels:
      app: nginx # has to match .spec.template.metadata.labels
  serviceName: "nginx"
  replicas: 3 # by default is 1
  minReadySeconds: 10 # by default is 0
  template:
    metadata:
      labels:
        app: nginx # has to match .spec.selector.matchLabels
    spec:
      terminationGracePeriodSeconds: 10
      containers:
      - name: nginx
        image: k8s.gcr.io/nginx-slim:0.8
        ports:
        - containerPort: 80
          name: web
        volumeMounts:
        - name: www
          mountPath: /usr/share/nginx/html
  volumeClaimTemplates:
  - metadata:
      name: www
    spec:
      accessModes: [ "ReadWriteOnce" ]
      storageClassName: "my-storage-class"
      resources:
        requests:
          storage: 1Gi
```

notes:

Este fichero `yaml` muestra las tres partes principales que forman un `StatefulSet`:

* El `Headless Service` utilizado para gestionar los nombres de los `Pods`
* El objeto `StatefulSet`. La especificación indica que se levatarán tres réplicas
  con un contenedor de `ngnix`
* Por último, el objeto `volumeClaimTemplates` que indica qué `PersistentVolume Provisioner` 
  se utilizará para asignar los volúmenes

note:

Se debe especificar el campo `.spec.selector`. Este campo debe corresponder con las
etiquetas definidas en `.spec.template.metadata.labels`. A partir de la versión 1.8, 
si no se especifica el campo `.spec.selector`, se producirá un error de validación
en el momento de creación del `StatefulSet`.

notes:

_Minimum ready seconds_ (en beta a partir de 1.23).

Número de segundos durante los cuales todos los contenedores de un `Pod` del `StatefulSet`
deben estar en ejecución sin fallar antes de que el `Pod` pueda considerarse 
como disponible.

El valor por defecto es 0 (el `Pod` se considerará disponible tan pronto como esté `Ready`)
